지도

지도

지도는 웹 매핑 응용 프로그램의 중심 부분이며의 인스턴스로 표시됩니다 OpenLayers.Map의 클래스입니다.

응용 프로그램은 하나 이상의 지도를 가질 수 있지만 지도 인스턴스에 첨부 된 레이어와 컨트롤이 다른 지도를 가질 수 없습니다

레이어

레이어

레이어는 우리가 지도에 표현하는 요소의 그룹/분류 입니다.

OpenLayers.Layer는 모든 레이어의 기본 클래스이며, 상속 된 모든 클래스에 대한 몇 가지 일반적인 속성을 정의:

  • name 모두 이름을 가지고 있습니다.(LayerSwitcher 컨트롤에서 사용됩니다.)
  • opacity 투명도를 결정하는 0..1 범위 값
  • projection 레이어에 포함 된 데이터의 투영
  • maxExtent 데이터의 최대 범위
  • resolutions 지도 해상도(지도 단위의 픽셀값)의 내림차순 목록

레이어의 계층 구조

우리는 레이어를 두 가지 카테고리로 분류 :

  • 래스터 (Imagery)
  • 벡터 (Features)

베이스 레이어 와 오버레이

OpenLayers는 레이어를 베이스 레이어 와 오버레이 두 종류로 분류 :

  • 베이스 레이어는 지도의 여러 측면을 제어합니다. (지도에서 사용하는 프로젝션 등)
  • 베이스 레이어는 함께 사용할 수 없습니다, 한 번에 하나만 표시 할 수 있습니다.
  • 여기에 임의의 수 만큼 오버레이 시킬 수 있습니다
  • OpenLayers.Layer에서 상속받은 isBaseLayer 속성으로 레이어의 종류를 결정합니다.
  • OpenLayers.Map 클래스의 allOverlays 속성으로 모든 레이어를 오버레이로 사용할 수 있습니다.

벡터 레이어

벡터 레이어

벡터 레이어는 데이터 소스에서 피쳐를 불러오는 일을 할 수 있습니다 ...

피쳐는 현실 세계에서 존재하는 것 들을 디지털로 표현 한 것: 도시, 강, 산, ...

참고 사항

피쳐

  • 피쳐는 속성을 가질 수 있습니다: 도시 이름, 인구 등
  • 피쳐는 시각적 스타일을 가질 수 있습니다: pointRadius, lineWidth, ... (OpenLayers.Feature.Vector.style 상수 참고)
  • OpenLayers.Feature 클래스는 피쳐의 기본을 정의합니다.
  • OpenLayers.Feature.Vector 클래스는 벡터 레이어에서 우리가 실제로 사용 하게 될 클래스입니다.

지오메트리

  • 피쳐는 지오메트리 인스턴스로 표현됩니다: 점, 선, 면, ... 이 인스턴스들은 OpenLayers.Geometry의 하위 클래스여야 합니다.

Note

  • 지오메트리는 피쳐의 스타일을 사용하여 표현되며, 없을 경우 에는 벡터 레이어의 스타일을 사용하여 표현합니다.
  • OpenLayers는 지리적 데이터의 일반적인 저장 모델인 Simple Feature Access 표준을 구현했습니다.

Renderers

브라우저에는 그래픽을 표현하는 다양한 기술이 존재합니다.(대부분 HTML5 캔버스 요소와 SVG)

렌더러는 추상 클래스이며, 브라우저에서 사용하는 기술을 제외한 벡터 레이어에서 피쳐(지오메트리)를 렌더링하는 방법을 담고 있습니다.

프로토콜 및 포맷

벡터 레이어는 많은 데이터 소스 (HTTP, WFS, SOS, ...) 에서 여러 데이터 포맷 (GeoJSON, GML, KML, ...) 으로 데이터를 로드 할 수 있습니다.

  • 프로토콜은 OpenLayers.Protocol의 일부 하위 클래스의 인스턴스여야 하며, 벡터 레이어가 데이터 소스와 통신하는 방식을 결정 합니다.
  • 포맷은 OpenLayers.Format의 일부 하위 클래스의 인스턴스 여야하며, 피쳐를 읽거나 작성하는 방식을 결정합니다.

Strategies는 벡터 레이어 일부 동작들에 대해 관찰 할 수 있습니다

정리해 봅시다 ...

레이어에 그리는 경우 :

  • 레이어가 데이터를 로드해야 할 경우 protocol을 호출합니다
  • protocol은 데이터 소스와 통신하고 데이터를 읽습니다
  • protocol은 데이터를 지정된 format을 사용하여 feature로 변환하고, 레이어에 추가합니다.
  • 레이어는 features 렌더링 작업을 renderer에게 맡깁니다.
  • renderer는 적합한 geometry와 style을 사용하여 각 피쳐를 그립니다

이벤트

이벤트는 응용 프로그램에서 신경과 같습니다 (동작 -> 반응)

  • HTML 요소는 트리거 이벤트를 가지고 있습니다: 클릭 , 마우스 입력 등
  • 대부분의 OpenLayers 구성 요소들은 트리거 이벤트를 가지고 있습니다::
    • OpenLayers.Map : move, zoomend, addlayer, ...
    • OpenLayers.Layer : loadstart,loadend, added , ...
    • OpenLayers.Layer.Vector : featuresadded, featureremoved,vertexmodified, ...
    각 OpenLayers 클래스에서 사용 가능한 이벤트를 알아보시려면 API 문서 (또는 소스 코드)를 확인하세요

...그리고 OpenLayers 구성 요소의 이벤트를 어떻게 구현 하나요?

  • OpenLayers 클래스의 대부분은 event속성을 가지고 있으며, 이는 OpenLayers.Events의 인스턴스 입니다
  • OpenLayers.Events는 리스너 및 트리거 이벤트를 등록 하고, 알림을 제공하는 헬퍼 클래스입니다
  • 이벤트가 OpenLayers.Event 인스턴스로 실행됩니다
